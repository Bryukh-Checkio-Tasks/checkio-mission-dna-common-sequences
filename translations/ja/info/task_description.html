デオキシリボ核酸は既知の全ての生命体と多くのウイルスの成長と機能に使われる遺伝子命令を符号化している分子です。 DNA分子は、ねじれたはしごによく似たお互いに巻きついた2つの鎖(strands)でできています。そして、糖とリン酸でできているはしごの両側は 塩基と呼ばれる窒素含有化学物質の横木でつながれています。 各鎖はヌクレオチドと呼ばれる類似した単位の繰り返しの直線的配置(a linear arrangement)です。 各ヌクレオチドはグアニン(G)、アデニン(A)、チミン(T)、シトシン(C)のいずれかの窒素含有核酸塩基(nucleobase)でできています。 糖-リン酸主鎖(backbone)に沿って配置される塩基の特定の順序はDNA配列(DNA sequences)と呼ばれます。 この配列によって、特定の生物をその固有の形質に造るために必要な正確な遺伝子命令が決まります。

(訳注: 生物学ではsequenceは配列と訳すようですが、情報学のarrayの配列と紛らわしいので以下では単に列と記します。) このミッションで、我々はDNA鎖を比較して、2つの鎖の最も長い共通の塩基部分列(base subsequence)を発見します。 部分列(subsequence)はオリジナルの列(sequence)の中で連続した位置を占めている必要はありません。文字は互いに様々な距離に配置することができます。 これらの鎖は文字"ACGT"でできている文字列で表現できます。 例えば鎖"ACGTC"と"ACTC"の最長共通列は"ACTC"です。

あなたには2つの鎖の記述が与えられて、最長の共通部分列(部分文字列でありません)を見つけなければいけません。 もし複数の長さの等しい「最長」列があったときは、これらを全て文字列にして返さなければいけません。 ここでこれらの列はコンマで区切った辞書順序で記述されます。 例えば、"CGCTA"と"TACCG"では3つの列"CC"と"CG"と"TA"があるので結果は"CC,CG,TA"になります。 もし与えられた列に共通の列がないときは空文字列を返します。

入力: 2つのパラメータ。塩基列、文字列。

出力: 共通の最長列または複数の列、文字列。
